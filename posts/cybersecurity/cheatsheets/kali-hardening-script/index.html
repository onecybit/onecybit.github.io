Kali Linux Hardening

  • kali
  • linux
  • hardening
  • ufw
  • fail2ban
  • clamav
  • lynis

A small Bash script that brings a fresh Kali install to a sane defensive baseline: applies updates, installs common defensive tooling, locks down inbound traffic, then runs three independent scans before printing a hardening report.

The original was published by Gerard King under the MIT License. This version is a cleanup with each tool documented inline, safer error handling (set -euo pipefail, root check up front), and a few practical fixes — e.g. stopping clamav-freshclam before running freshclam so the daemon's DB lock does not fail the update.

Download automate_cybersecurity_kali.sh

Run it as root on a host you own:

chmod +x automate_cybersecurity_kali.sh
sudo ./automate_cybersecurity_kali.sh

What it installs

ToolPurpose
ufwFriendly front-end for nftables/iptables. Enforces a default-deny inbound policy.
fail2banWatches service logs (e.g. SSH) and dynamically bans IPs that fail authentication repeatedly.
rkhunterRootkit Hunter. Scans for known rootkit signatures, suspicious permissions, hidden files, and altered system binaries.
chkrootkitIndependent second-opinion rootkit scanner. Different signature set and detection heuristics.
clamavOpen-source antivirus engine. Bundled with freshclam (signature updater) and clamscan (on-demand scanner).
lynisHost-hardening auditor. Produces a prioritised report with a numeric "hardening index".

What it changes

  • UFW set to deny inbound by default, allow outbound, permit OpenSSH.
  • fail2ban enabled on boot.
  • Updates and runs rkhunter, clamscan against /home, and lynis audit system --quick.
  • Logs land in the usual places: /var/log/rkhunter.log, /var/log/lynis.log, /var/log/lynis-report.dat.

The script

#!/usr/bin/env bash
#
# automate_cybersecurity_kali.sh — Defensive setup script for Kali Linux
#
# Hardens a Kali Linux host: applies updates, installs common defensive tools,
# enables a host firewall and intrusion-prevention, and runs malware/rootkit
# scanners and a system auditor. Intended for the operator's own machine.
#
# Original author: Gerard King — https://www.gerardking.dev
# License:         MIT (preserved below)
# Cleaned and annotated by onecybit — https://onecybit.github.io
#
# Usage: sudo ./automate_cybersecurity_kali.sh

set -euo pipefail

# Every step below needs root (apt, systemctl, ufw, freshclam, etc.).
if [[ $EUID -ne 0 ]]; then
    echo "Run as root: sudo $0" >&2
    exit 1
fi

log() { printf '\n[+] %s\n' "$*"; }


# ── 1. System update ────────────────────────────────────────────────────────
# Refreshes the apt package index and applies pending security/feature
# upgrades, so every tool installed below is at its latest patched version.
update_system() {
    log "Updating package index and upgrading installed packages…"
    apt-get update
    DEBIAN_FRONTEND=noninteractive apt-get upgrade -y
}


# ── 2. Install defensive tooling ────────────────────────────────────────────
# ufw         Uncomplicated Firewall — a friendly front-end for nftables/iptables.
# fail2ban    Log-watcher / IP-banner. Tails service logs and dynamically
#             inserts firewall rules to block hosts that show repeated failed
#             authentication attempts.
# rkhunter    Rootkit Hunter — scans for known rootkit signatures, suspicious
#             file permissions, hidden files, and altered system binaries.
# chkrootkit  Independent second-opinion rootkit scanner. Pairs with rkhunter.
# clamav      Open-source antivirus engine. Ships with `freshclam` (signature
#             updater) and `clamscan` (on-demand scanner).
# lynis       Host-hardening auditor. Produces a prioritised report with a
#             numeric "hardening index".
install_security_tools() {
    log "Installing defensive tooling…"
    apt-get install -y ufw fail2ban rkhunter chkrootkit clamav lynis
}


# ── 3. Firewall ─────────────────────────────────────────────────────────────
# Default-deny inbound is the standard hardening posture. SSH is allowed so
# the operator does not lock themselves out of a remote box — drop the
# `ufw allow OpenSSH` line if the host is purely local.
configure_ufw() {
    log "Configuring UFW (deny in, allow out, permit SSH)…"
    ufw default deny incoming
    ufw default allow outgoing
    ufw allow OpenSSH
    ufw --force enable
}


# ── 4. fail2ban ─────────────────────────────────────────────────────────────
# Default jail bans an IP after 5 failed SSH auth attempts for 10 minutes.
# Tune /etc/fail2ban/jail.local to change thresholds or add jails.
setup_fail2ban() {
    log "Enabling fail2ban service…"
    systemctl enable --now fail2ban
}


# ── 5. Rootkit Hunter ───────────────────────────────────────────────────────
# `--update` refreshes the signature database (kept non-fatal).
# `--check --sk` runs every test and skips the interactive prompts so the
# script can run unattended. Output: /var/log/rkhunter.log
run_rkhunter() {
    log "Running rkhunter…"
    rkhunter --update || true
    rkhunter --check --sk
}


# ── 6. ClamAV ───────────────────────────────────────────────────────────────
# `freshclam` cannot run while the clamav-freshclam daemon holds the DB lock,
# so we briefly stop it. `clamscan -r --bell -i /home` recursively scans the
# user's home directory and prints only infected files.
run_clamav() {
    log "Updating ClamAV signatures…"
    systemctl stop clamav-freshclam 2>/dev/null || true
    freshclam || true
    systemctl start clamav-freshclam 2>/dev/null || true

    log "Scanning /home for malware…"
    clamscan -r --bell -i /home
}


# ── 7. Lynis audit ──────────────────────────────────────────────────────────
# Writes /var/log/lynis.log and /var/log/lynis-report.dat with concrete
# remediation suggestions and a hardening index. `--quick` skips pauses.
run_lynis() {
    log "Running lynis system audit…"
    lynis audit system --quick
}


main() {
    log "Starting cybersecurity setup on Kali Linux…"
    update_system
    install_security_tools
    configure_ufw
    setup_fail2ban
    run_rkhunter
    run_clamav
    run_lynis
    log "Cybersecurity setup on Kali Linux completed."
}

main "$@"

Notes

  • Drop the ufw allow OpenSSH line if the host is purely local — it is there so remote operators do not lock themselves out.
  • Tune /etc/fail2ban/jail.local for different ban thresholds, or to add jails for other services.
  • Read the Lynis report — it ranks suggestions; address [!] warnings before tweaking lower-priority hints.
